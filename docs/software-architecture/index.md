# 🧱 Ruta Software Architect

<span className="badge-arch">Software Architecture</span>

Esta ruta está diseñada para pasar de implementar features a **diseñar sistemas**: claros en su dominio, resilientes en producción y sostenibles para equipos grandes.

![Arquitectura de sistemas](https://upload.wikimedia.org/wikipedia/commons/1/1b/Client-server-model.svg)

## ¿Qué hace un Software Architect?

Un/a Software Architect:

- define límites entre componentes y equipos;
- toma decisiones de trade-off (latencia, costo, complejidad, time-to-market);
- establece estándares de diseño, calidad y operación;
- reduce riesgo técnico en decisiones de largo plazo.

## Roadmap por fases

| Fase | Duración sugerida | Objetivo |
|---|---|---|
| **Phase 1 — Foundations** | 4–8 semanas | Entender atributos de calidad y patrones base |
| **Phase 2 — Modeling & Boundaries** | 6–10 semanas | Modelar dominio y contratos entre servicios |
| **Phase 3 — Data & Scale** | 6–10 semanas | Diseñar consistencia, escalabilidad y confiabilidad |
| **Phase 4 — Security & Operations** | Continuo | Asegurar entrega segura, gobernanza y evolución |

## Cómo estudiar esta ruta

1. Empieza por `system-design-principles` y aterriza cada concepto en tu sistema actual.
2. En cada módulo, implementa al menos **1 ADR** (Architecture Decision Record).
3. Cierra cada fase con un mini-proyecto de rediseño sobre un caso real.

## Resultado esperado

Al finalizar, podrás:

- diseñar un sistema end-to-end (desde dominio hasta operación);
- justificar decisiones de arquitectura con métricas y riesgo;
- guiar equipos para mantener una arquitectura evolutiva.
